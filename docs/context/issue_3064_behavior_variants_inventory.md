# Issue #3064 Behavior Variants Inventory

Date: 2026-06-18

Related issue: <https://github.com/ll7/robot_sf_ll7/issues/3064>

Status: preflight/inventory evidence; not a full benchmark campaign or paper-facing result.

## Goal

Issue #3064 asks which native and adapter-backed pedestrian/social-force behavior variants are
valid benchmark inputs versus diagnostic-only or unavailable models. This note records the first
bounded fail-closed classification so later robot-influence and robustness campaigns can select
behavior variants without silently upgrading fallback or missing-runtime rows.

## Result

The current preflight classifies six variants:

| Variant | Classification | Interpretation |
| --- | --- | --- |
| `social_force` | `benchmark_valid_candidate` | Current native Robot SF Social Force baseline candidate. |
| `ammv_social_force` | `diagnostic_only` | Existing same-seed adapter-mode evidence found no default-vs-AMMV frame or episode metric delta. |
| `social_navigation_pyenvs_orca` | `not_available` | External checkout missing in this worktree. |
| `social_navigation_pyenvs_socialforce` | `not_available` | External checkout missing and pinned `socialforce==0.2.3` backend missing. |
| `social_navigation_pyenvs_sfm_helbing` | `not_available` | External checkout missing in this worktree. |
| `social_navigation_pyenvs_hsfm_new_guo` | `not_available` | External checkout missing in this worktree. |

The tracked summary reports status counts of one `benchmark_valid_candidate`, one
`diagnostic_only`, and four `not_available` rows. Under
[issue_691_benchmark_fallback_policy.md](issue_691_benchmark_fallback_policy.md), the diagnostic
and not-available rows must not count as benchmark-success evidence.

## Follow-Up Boundary

To upgrade any Social-Navigation-PyEnvs row, stage or hydrate the upstream checkout with provenance,
install the required runtime dependency for `social_navigation_pyenvs_socialforce`, rerun the
preflight, and then run the relevant adapter probe before any benchmark-strength comparison. To
upgrade AMMV, use a scenario or direct mechanism trace path that produces an observed
default-vs-AMMV behavioral delta; repeating the current adapter-mode sweep is unlikely to add value.
