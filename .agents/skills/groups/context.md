# Context-Oriented Skills

Use these skills when the task primarily needs orientation, handoff context, or documentation-safe routing.

- `benchmark-overview` for benchmark-faithful orientation before analysis or edits.
- `context-map` for focused repository surface discovery before multi-file work.
- `experiment-context` for config-first training/evaluation lineage and validation gates.
- `planner-integration` for planner-family feasibility and provenance boundaries.
- `paper-facing-docs` for claim-sensitive benchmark or manuscript-support docs.
- `review-benchmark-change` for semantic benchmark review.
- `context-note-maintainer` for durable `docs/context/` notes.

Validation-owned skills such as `implementation-verification` now live in the validation routing group, not this context note.
