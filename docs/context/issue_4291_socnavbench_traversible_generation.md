## What This Adds

`scripts/tools/generate_socnavbench_traversible.py`: a reproducible wrapper that builds the derived
SocNavBench traversible `traversibles/<MAP>/data.pkl` from a staged per-map mesh, with fail-closed,
CI-safe input validation. See section 7 of `docs/socnav_assets_setup.md` for the runbook.

## Why It Is Shaped This Way

- **Derived data, never committed.** The official SocNavBench package ships S3DIS *area*
  traversibles but not the custom-map ones (`ETH`/`Hotel`/`Univ`/`Zara`/`DoubleHotel`); SocNavBench
  generates those from each map's curated mesh. The output is generated data, so it is written into
  the data root next to the mesh and is regenerated by external users the same way.
- **CI-safe by construction.** Preflight (`--dry-run`, skip-if-absent) imports only the standard
  library plus the repo's external-data path resolver, reusing the `socnavbench-s3dis-eth` registry
  id so paths stay consistent with `manage_external_data.py` and `validate_socnav_map_batch.py`. The
  heavy SocNavBench mesh pipeline (`dotmap`, `mp_env`, swiftshader) is imported lazily and only on an
  explicit build. `dotmap` is confirmed absent from the main environment, so the module must — and
  does — import and test without it.
- **Fail closed.** A missing mesh exits `2` (matching `validate_socnav_map_batch.py`) with an
  actionable message naming the expected mesh path. An empty mesh directory is treated as *not
  staged* (placeholder shell), mirroring `prepare_socnav_assets.py`.

## Build-Path Assumption (maintainer confirms on first real run)

The vendored SocNavBench tree's single renderer-params factory is commented out
(`create_base_params` does not exist), and no in-repo code constructs `SBPDRenderer` to build a
traversible. `_build_socnav_traversible` therefore reconstructs a flat `DotMap` carrying exactly the
attributes `SBPDRenderer.__init__` / `get_config` read (`dataset_name`, `building_name`,
`robot_params`, `flip`, `sbpd_data_dir`, `traversible_dir`, `camera_params.modalities`,
`load_meshes`, `load_traversible_from_pickle_file`), forcing the `occupancy_grid` modality so the
mesh-derived traversible is built without the RGB/depth GL renderer. This is the defensible
invocation given the vendored tree; the maintainer confirms it on the first environment run with the
ETH mesh staged. The wrapper's preflight/skip/idempotency behavior is fully covered by tests
regardless.
