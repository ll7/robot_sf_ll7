## Goal

Finish the dormant SICNav integration to smoke level: prove the existing wrapper works against
the real upstream on the open-source dependency path, and measure per-`step()` solve time vs the
benchmark step budget. NO campaign, NO tier promotion, NO roster addition.

## Contract reconciliation (the key finding)

The wrapper previously assumed a `SICNavPolicy(checkpoint_path=, solver=, device=)` constructor
plus `select_action(obs)`, and the metadata named `sicnav_diffusion.policy.sicnav_acados.SICNavAcados`
as the upstream class. The **real** upstream API is a `policy_factory` registry, and:

* `sicnav_diffusion.policy.sicnav_acados.SICNavAcados` requires **Acados** (+ the JMID diffusion
  forecaster) — that is the path the issue says to skip.
* `sicnav.policy.campc.CollisionAvoidMPC` is the **CasADi + IPOPT** MPC (`import casadi as cs`;
  uses `orca_c_wrapper` → `rvo2`). With `hum_model=orca_casadi_kkt` and `priviledged_info=false`
  this is the SICNav-np variant. This is the open-source path.

So the wrapper was reconciled (wrapper-side only, `robot_sf/baselines/sicnav.py`) to drive
`CollisionAvoidMPC` through its real API: `configure(configparser)` → `set_env(adapter)` →
`predict(FullState + humans)` → `ActionRot(v, r)`, mapped back to a Robot SF unicycle
`{"v", "omega"}` action (`omega = action.r / dt`, since upstream `ActionRot.r` is the angular-rate
control integrated over one step). The legacy generic `SICNavPolicy`/`load_policy` fallback is
preserved for synthetic/legacy repos.

## Smoke result: PASS

`tests/baselines/test_external_mpc_wrappers.py::test_sicnav_campc_step_runs_against_staged_upstream`
drives `SICNavPlanner.step()` against the staged upstream on the open-source path and asserts a
valid, finite, clamped `{"v", "omega"}` action plus `get_metadata()["status"] == "ok"`. It skips
cleanly when the staged checkout or the `casadi`/`rvo2`/`gym` stack is absent (CI default), so it
does not regress environments without the heavy dependencies.

## Per-step solve time (the decisive measurement)

Measured through `SICNavPlanner.step()` (the benchmark entrypoint) on synthetic Observations;
6 steps per neighbor count (first = CasADi compile + first solve warmup; remaining 5 = steady-state
warmstarted solves). Benchmark step budget = `SimConfig.time_per_step_in_secs` = **100 ms**.

| Neighbors | First step (ms) | Steady mean/step (ms) | Steady max (ms) | × budget |
| ---: | ---: | ---: | ---: | ---: |
| 2  | 883   | 197  | 202  | 2.0×  |
| 5  | 2520  | 506  | 518  | 5.1×  |
| 10 | 11077 | 1454 | 1535 | 14.5× |

Raw results: `output/issue_4870/sicnav_timing.json` (git-ignored, regenerated by the reproducer);
reproducer: `scripts/tools/measure_sicnav_timing.py`.

Timing verdict: the steady-state solve is **above** the 100 ms real-time step budget at every
neighbor count and scales roughly linearly with neighbor count. The redistributable open-source
path (CasADi/IPOPT, no HSL, no compiled Acados) is therefore **campaign-infeasible for real-time**.
This negative result is the deliverable.

Caveat to carry forward: SICNav models pedestrians via ORCA; the Robot SF default crowd is
social-force. Any future comparison slice must use the ORCA pedestrian mode and frame the
comparison accordingly. Static-obstacle mapping (Robot SF obstacle geometry → upstream
`static_obs`) is not wired in this smoke; the smoke passes empty obstacles.

## Recommendation (separate maintainer call)

Do NOT rerun the uncompiled CasADi/IPOPT path in a benchmark. The gate-doc `sicnav` row is
updated to point here. Revisit only with (a) a compiled Acados + HSL build or a horizon/solver
change that brings per-step solve under the 100 ms budget, and (b) the ORCA pedestrian crowd.
Roster/campaign decisions remain a separate maintainer call after that.
