# Issue #3482 — Metric-Semantics Reconciliation Table

**Status:** integrity infrastructure. Core `EpisodeEventLedger.v1` reconciliation guards exist in
`robot_sf/benchmark/event_ledger.py` and are property-tested in
`tests/benchmark/test_event_ledger.py`.

Those guards cover:

- exact collision implies collision metric greater than zero;
- `goal_reached` and `invalid_run` are mutually exclusive;
- every ledger metric has a definition and version.

The reconciliation export and continuous-integration guard are covered by
`scripts/benchmark/export_event_ledger_reconciliation.py`,
`scripts/dev/check_event_ledger_reconciliation_guard.sh`, and
`tests/validation/test_event_ledger_reconciliation_guard.py`.

## What

`robot_sf/benchmark/event_ledger_reconciliation.py` builds one
`event_ledger_reconciliation.v1` metric-semantics reconciliation table from one
`EpisodeEventLedger.v1` payload.

For each reported metric field, the table records:

- `field`;
- `level`;
- `kind` (`exact_or_sampled` or `derived`);
- `producer`;
- representative downstream consumers;
- overall audit result and reconciliation violations from `reconcile_event_ledger`.

Representative consumers are intentionally conservative. They provide audit orientation, not an
exhaustive dependency graph.

## Checked-In Guard Evidence

Small fixture-level guard evidence is tracked under
`docs/context/evidence/issue_3482_event_ledger_reconciliation_guard/`:

- `reconciled_fixture_reconciliation.jsonl` records two clean fixture rows generated by
  `export_event_ledger_reconciliation.py --fail-on-violations`.
- `violating_fixture_reconciliation.jsonl` records the regression fixture where an exact collision
  event coexists with zero collision count. The exporter writes the violation row and exits `1`.
- `manifest.json` records the source fixtures, generator, hashes, expected exit codes, and claim
  boundary.

This bundle is intentionally small and reviewable. It is not raw benchmark output.

## Scope Boundary

The reconciliation table and guard are integrity infrastructure. The remaining broader Definition
of Done item, re-running frozen `0.0.2` traces through the corrected analyzer and emitting a
before/after numerical diff, needs archived release data and should remain separate from this
fixture-level guard evidence.

Until that rerun is available, this note does not change any benchmark, paper, or dissertation
claim.

## Tests

`tests/benchmark/test_event_ledger_reconciliation.py` validates that the table emits one row per
reported field, records valid kind/producer metadata, names the collision metric producer as
exact-or-sampled, reconciles clean ledgers, and surfaces exact-collision-vs-zero-metric
disagreements instead of silently resolving them.

`tests/validation/test_event_ledger_reconciliation_guard.py` validates that the CI-facing guard
passes on the reconciled fixture, fails closed on the violating fixture, and remains wired into
`scripts/dev/ci_driver.sh`.
